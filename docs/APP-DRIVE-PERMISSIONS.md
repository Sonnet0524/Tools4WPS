# 应用身份云文档权限申请指南

## 🔑 需要的权限

使用**应用身份**访问云文档，需要申请以下权限：

### 1. **驱动盘权限**
- **权限名**: `kso.drive.readwrite`
- **用途**: 创建应用盘
- **授权类型**: 应用授权

### 2. **文件管理权限**
- **权限名**: `kso.file.readwrite`
- **用途**: 在应用盘中管理文件
- **授权类型**: 应用授权

### 3. **应用文件权限**（关键）
- **权限名**: `kso.appfile.readwrite`
- **用途**: 应用身份管理自己的文件
- **授权类型**: 应用授权
- **说明**: 这是专门给应用身份使用的权限

### 4. **多维表格权限**（如需要）
- **权限名**: `kso.dbsheet.read`, `kso.dbsheet.readwrite`
- **用途**: 管理多维表格
- **授权类型**: 应用授权 ✓

---

## 📝 申请步骤

### 步骤1: 登录开放平台
访问 https://open.wps.cn

### 步骤2: 进入应用详情
1. 点击「应用管理」
2. 选择你的应用
3. 进入「权限管理」页面

### 步骤3: 申请权限
找到以下权限并申请：

```
✅ kso.drive.readwrite       (管理驱动盘)
✅ kso.file.readwrite        (管理文件)
✅ kso.appfile.readwrite     (应用身份管理文件)  ⭐ 关键
✅ kso.file.search           (搜索文件)           [可选]
✅ kso.file_link.readwrite   (文件分享)           [可选]
```

### 步骤4: 等待审批
- 提交后等待企业管理员审批
- 审批通过后即可使用

## 💡 关键区别

### **kso.file.readwrite vs kso.appfile.readwrite**

**kso.file.readwrite**:
- 用于操作用户盘/团队盘中的文件
- 需要用户授权或特定场景

**kso.appfile.readwrite**:
- 用于操作**应用盘**中的文件
- 专属于应用身份
- 应用自己管理自己的文件

**实际使用中**:
- 创建应用盘 → 需要 `kso.drive.readwrite`
- 在应用盘创建文件 → 需要 `kso.appfile.readwrite`
- 建议同时申请两个权限

## 🔧 常见问题

### Q: 已经有 kso.file.readwrite，为什么还403？
**A**: `kso.file.readwrite` 主要用于操作用户可见的文件，应用盘需要使用 `kso.appfile.readwrite`。

### Q: 应用盘和用户盘有什么区别？
**A**: 
- **应用盘**: 属于应用，无UI界面，仅API访问
- **用户盘**: 属于用户，有云文档界面

### Q: 多维表格需要用户授权吗？
**A**: 不需要！多维表格API支持应用授权，可以直接使用。

### Q: 应用盘有存储空间限制吗？
**A**: 有，创建时可以指定配额（total_quota），单位为字节。

## ✅ 申请模板

给管理员看的权限申请说明：

```
应用名称: [你的应用名]
应用ID: AK20260308LGOUTU

申请权限:
1. kso.drive.readwrite - 创建应用专属存储盘
2. kso.appfile.readwrite - 管理应用自己的文件

用途说明:
- 应用需要创建独立的存储空间
- 存储应用配置、模板文件等
- 不占用用户个人空间
- 通过API管理，无界面展示

权限范围:
- 仅应用自己创建的盘和文件
- 不影响用户个人数据
```
