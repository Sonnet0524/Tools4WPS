## 遇到的问题

### 问题1：URL结构错误导致爬取失败

**现象**：
- 最初爬取的文档大部分只包含标题，显示"您访问的文档不存在"
- 24份文档中，大部分只有10-15行

**原因**：
- 使用了猜测的URL结构
- 错误URL：`/guide/start/intro`
- 正确URL：`/guide/start/overview`

**解决方案**：
1. 从成功访问的页面提取实际链接
2. 使用`extract-links.js`工具提取真实URL
3. 基于实际URL重新爬取

**结果**：
- 成功爬取59份文档
- 25份详细文档（>50行）
- 7份超详细文档（>100行）

---

### 问题2：SPA页面动态内容加载

**现象**：
- WPS开放平台使用SPA架构
- 部分内容通过JavaScript动态加载
- 简单的HTTP请求无法获取完整内容

**解决方案**：
1. 使用Puppeteer浏览器自动化工具
2. 创建专门的`fetch-spa.js`工具
3. 等待足够时间（5秒）确保内容加载
4. 滚动页面触发懒加载
5. 尝试多种内容选择器

**关键代码**：
```javascript
// 等待动态内容加载
await new Promise(resolve => setTimeout(resolve, 5000));

// 滚动页面触发加载
await page.evaluate(async () => {
  await new Promise((resolve) => {
    // 滚动逻辑
  });
});

// 尝试多种选择器
const selectors = [
  'article',
  '.markdown-body',
  '.documentation-content',
  // ...更多选择器
```

## 有效做法

### 1. 工具化思维

创建可复用的工具，而不是一次性脚本：
- **extract-links.js**: 从页面提取所有链接
- **fetch-spa.js**: 爬取SPA页面完整内容
- **fetch-page.js**: 基础页面抓取

**好处**：
- 可重复使用
- 可持续改进
- 可固化到项目中

### 2. 递归提取策略

不要猜测URL，而是从实际页面提取：
1. 从学习地图开始（第0层）
2. 提取所有链接（第1层）
3. 访问每个链接继续提取（第2层）
4. 去重，生成完整URL清单

### 3. 分层优先级管理

将文档按优先级分类：
- **P0**: 必须立即爬取（核心流程、认证授权）
- **P1**: 重要文档（API详细说明、开发指南）
- **P2**: 补充文档（扩展功能、参考文档）

### 4. 质量检查机制

每个文档爬取后自动检查：
- 行数统计
- 内容质量标记
- 错误检测（"文档不存在"等）

## 经验教训

1. **不要猜测URL**：从实际页面提取链接，不要假设URL结构
2. **工具要可复用**：创建通用工具，而不是一次性脚本
3. **质量检查很重要**：及时发现问题，避免大量返工
4. **文档组织要清晰**：良好的结构比数量更重要
5. **Team分工要明确**：专门的Team负责专门的任务
