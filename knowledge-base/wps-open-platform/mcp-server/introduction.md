MCP 模型上下文协议
简介

MCP（Model Context Protocol，模型上下文协议）是连接大语言模型与外部工具的标准协议框架，像智能交互的“万能接口”，能让模型任意使用丰富的外部工具，让使用者通过自然语言的方式触发AI+工具能力，解决复杂场景问题。

WPS 开放平台基于WPS 365 开放API的能力兼容MCP协议，陆续完成适配接入，支持企业以通用化的方式在智能体中调用365 API，满足企业AI业务开发需求。

企业可填写(表单（WPS 365 MCP server试用申请）)申请开通权限。

特性

AI时代的“万能接口”

适配多种场景、兼容各类 AI 模型与外部工具的数据交互枢纽

标准化协议打通大模型与数据工具

通过统一规范的技术协议，消除大模型与各类数据工具之间的兼容性壁垒

自然语言触发「AI + 工具」新范式

开口即能调用 AI 模型联动各类工具，让数据查询、设备控制等操作通过自然语言指令一键激活

助力开发者高效构建智能服务

提供标准化协议与模块化工具链，开发者无需从零搭建复杂接口

技术架构

MCP 采用模块化的客户端-服务器架构，解耦了 AI 助手与后端服务之间的关系，典型的 MCP 部署包括：

主机进程（Host）：运行大模型的应用程序，连接用户界面
MCP 客户端（Client）：负责发送请求和接受响应，并处理协议转换
MCP 服务器（Server）：通过标准化的 MCP 协议，提供各方工具的能力
工作原理

当用户以自然语言提出请求时：

MCP 解析请求并调用触发工具
工具访问资源后将结果返回给客户端
客户端将结果传给大模型
大模型结合上下文生成自然语言输出

用户无需感知全过程即可进行流畅的自然语言交互。

核心能力

打破模型能力边界，通过标准化协议，MCP 让大模型无需内置所有工具逻辑，只需通过协议动态调用外部能力，实现 “自然语言指令→工具执行→结果解析” 的闭环。

应用场景
智能文档处理 员工在撰写工作文档时，通过 MCP Server，大语言模型可自动从企业内部知识库、过往文档中提取相关数据与案例，辅助内容创作，同时进行语法检查、格式优化，大幅提升文档撰写效率与质量。
高效项目管理 项目经理可使用 WPS MCP Server，让大语言模型连接项目管理工具、团队成员日程安排以及项目进度数据库。
智能客户服务 通过AI+MCP搭建完全属于企业自身的专属客服，通过快速检索企业产品知识库、客户历史记录等，提高传统AI客服助手的问题解决率。
办公流程自动化 对于重复性办公流程，如费用报销审批、合同签订流程等，利用WPS MCP Server，大语言模型可自动触发相关办公工具，完成信息填写、流程流转等操作，实现办公流程自动化，减少人工干预，提升办公效率。
