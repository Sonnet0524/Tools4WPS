# 导入假勤审批记录

用于将三方系统（如企业自有OA）审批结果数据回写到轻打卡应用中，进行假勤数据的同步。避免在三方审批系统中请假后，轻打卡应用中无请假信息而导致考勤异常的问题



**标签**：`打卡假勤审批` 


## 请求说明
| 字段          | 值                  |
|--------------|---------------------|
| **<div style="white-space: nowrap;">请求地址</div>**  | https://openapi.wps.cn/v7/attendance/approval_records/import          |
| **<div style="white-space: nowrap;">HTTP 方法</div>** | `POST`    |
| **<div style="white-space: nowrap;">接口描述</div>**  | 用于将三方系统（如企业自有OA）审批结果数据回写到轻打卡应用中，进行假勤数据的同步。避免在三方审批系统中请假后，轻打卡应用中无请假信息而导致考勤异常的问题 |
| **<div style="white-space: nowrap;">签名方式</div>**  | [KSO-1](https://open.wps.cn/documents/app-integration-dev/wps365/server/api-description/signature-description) |
| **<div style="white-space: nowrap;">限频策略</div>** | 无 |
| **<div style="white-space: nowrap;">权限要求</div>** | <div><div>管理假勤审批记录(应用授权) `kso.attendance_approval_record.readwrite`</div></div> |
