# 会议业务域概述

## 会议介绍

会议 API 基于 WPS 会议功能开放了对会议、录制、纪要等资源的操作与查询能力。你能以应用或用户的身份调用会议 API 来实现多种功能。例如：

- 查询会议信息，管理参会人和主持人
- 控制会议录制的开始和停止
- 获取录制和纪要的智能处理结果（语音转写、AI总结、章节划分）
- 导入外部录制文件生成纪要
- 订阅会议相关事件

**资源关系说明：**

- **会议**：在线视频会议的核心资源，代表每一场实际开始的会议，包含会议 ID、主题、参会人、时间等基本信息。
- **参会人**：会议的参与者，包括主持人和普通参会人。
- **会议录制**：开启录制后，在会议过程中产生的音视频录制文件，支持语音转写、章节划分、AI 总结等智能处理。
- **会议纪要**：开启AI纪要后，基于会议中的音频流，进行语音转写、AI 总结等智能处理；除了每场自动生成外，也可以导入外部录制文件生成。

一个会议预约可以有多场会议。一场**会议**可以有多个**参会人**，产生多个**会议录制**文件和**会议纪要**。**会议录制**和**会议纪要**都支持智能化处理，提取会议关键信息。

## 业务模块

### 会议管理

查询会议信息、管理参会人员。支持获取会议列表和详细信息、邀请或移除参会人员、设置主持人、结束会议。

[查看会议管理详细文档 →](/app-integration-dev/wps365/server/meeting/meetings/introduce)

### 会议录制

控制录制开始和停止，获取录制文件列表和详情，以及获取录制的智能化处理结果（语音转写、章节划分、AI 总结）。

[查看会议录制详细文档 →](/app-integration-dev/wps365/server/meeting/recordings/introduce)

### 会议纪要

除了会议中自动生成的纪要外，还支持导入外部录制文件生成纪要，并提供智能化处理能力（语音转写、AI 总结）。支持获取、导入、删除会议纪要。

[查看会议纪要详细文档 →](/app-integration-dev/wps365/server/meeting/minutes/introduce)

### 事件订阅

支持订阅以下事件：

- **企业会议变更**：会议开始、结束等状态变化
- **会议录制变更**：录制完成并可用
- **会议纪要变更**：纪要生成完成
- **导入纪要状态变更**：导入纪要的处理状态更新

## 典型应用场景

### 1. 报障系统自动会议与记录

企业报障系统在收到故障工单后，自动组织会议讨论并记录全过程：
- 报障工单触发后，通过日历接口创建会议并邀请相关技术人员
- 会议开始时，调用开始录制接口自动开启录制功能
- 会议进行中，根据需要调用邀请参会人接口拉更多专家进来协助
- 会议结束后，通过获取录制列表接口获取录制文件，调用获取录制语音转写、获取录制总结要点、获取录制章节等接口提取会议内容
- 将会议记录（语音转写、AI总结、章节）自动发送到报障群，作为问题处理的完整记录

### 2. 会议全流程自动化管理

将会议数据自动同步到业务系统，实现会议全流程跟踪：
- 订阅企业会议变更事件，实时获取会议开始、结束等状态变化
- 会议开始时，通过获取会议详情接口获取会议信息，向相关人员发送通知
- 会议进行中，可以通过邀请参会人、移除参会人、设置主持人等接口动态管理参会人员
- 会议结束后，调用获取会议详情接口获取完整的会议信息和参会人列表，同步到 OA、项目管理等业务系统
