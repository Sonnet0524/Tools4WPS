# 导入录制文件生成纪要

通过提供外部录制文件的语音转写内容下载地址来生成独立的会议纪要。此接口适用于您使用第三方录制系统，需要将录制内容接入 WPS 平台进行 AI 处理的场景。生成的纪要不关联 WPS 内部会议。

**注意：** 当前版本必须提供语音转写内容下载地址（`transcript_content_url`），语音转写内容需为 JSON 格式。

## 请求说明

| **请求地址** | **https://openapi.wps.cn/v7/minutes/create**                                                                                                                           |
| :----------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **请求方法** | POST                                                                                                                                                                        |
| **签名方式** | [KSO-1](/app-integration-dev/wps365/server/api-description/signature-description)                                                                                         |
| **权限要求** | 查询和管理会议室（应用授权） `kso.meeting_minutes.readwrite` |

## 请求头（Header）

| **Header 名称**     | **参数类型** | **是否必填** | **说明**                                                                                                     |
| :------------------ | :----------- | :----------- | :----------------------------------------------------------------------------------------------------------- |
| Content-Type        | string       | 是           | 使用：`application/json`                                                                                     |
| X-Kso-Date          | string       | 是           | RFC1123 格式的日期，例: `Wed, 23 Jan 2013 06:43:08 GMT`                                                      |
| X-Kso-Authorization | string       | 是           | KSO-1 签名值，详见[《签名方法》](/app-integration-dev/wps365/server/api-description/signature-description) |
| Authorization       | string       | 是           | 授权凭证，格式为：`Bearer {access_token}`                                                                    |

## 请求体（Body）

<!-- 为了保持文档格式统一：（1）参数说明中请使用中文全角标点符号；（2）中英文间请添加空格 -->

| **名称**                  | **参数类型** | **是否必填** | **说明**                                    |
|:------------------------|:---------|:---------|:------------------------------------------|
| ai_enable               | boolean  | 是        | 是否启用 AI                                  |
| duration                | integer  | 是        | 录制时长（单位毫秒）                                |
| recording_content_url   | string   | 是        | 录制内容下载地址（音视频文件）                           |
| recording_thumbnail_url | string   | 否        | 录制缩略图下载地址                                 |
| start_time              | integer  | 是        | 录制开始时间（单位秒）                               |
| title                   | string   | 是        | 录制标题                                      |
| transcript_content_url  | string   | 是        | 语音转写内容下载地址，下载内容需为 JSON 格式，详见下方「语音转写格式说明」 |

## 语音转写格式说明

`transcript_content_url` 指向的 JSON 内容格式要求如下：

**格式粒度：** 句子级别

```json
{
  "speakers": [
    {
      "name": "发言人姓名",
      "avatar": "发言人头像URL（可选）",
      "id": "发言人唯一标识（可以等于 name）"
    }
  ],
  "sentences": [
    {
      "speaker_id": "发言人ID",
      "paragraph_id": "段落ID",
      "start_time": 1000,
      "end_time": 5000,
      "text": "发言内容"
    }
  ]
}
```

**字段说明：**

- `speakers`: 发言人列表
  - `name`: 发言人姓名
  - `avatar`: 发言人头像URL（可选）
  - `id`: 发言人唯一标识，可以与 name 相同
- `sentences`: 句子列表（句子粒度）
  - `speaker_id`: 发言人ID，对应 speakers 中的 id
  - `paragraph_id`: 段落ID
  - `start_time`: 发言开始时间（单位毫秒），相对媒体时间
  - `end_time`: 发言结束时间（单位毫秒），相对媒体时间
  - `text`: 发言内容

## 请求地址示例

```
[POST] https://openapi.wps.cn/v7/minutes/create
```

## 请求体示例


## 响应体

<!-- 为了保持文档格式统一：（1）参数说明中请使用中文全角标点符号；（2）中英文间请添加空格 -->

| **名称**     | **参数类型** | **说明**                                                                                    |
|:-----------|:---------|:------------------------------------------------------------------------------------------|
| code       | integer  | 响应代码。非 0 表示失败，参照[《状态码说明》](/app-integration-dev/wps365/server/api-description/errorcode) |
| msg        | string   | 响应信息                                                                                      |
| data       | object   | 响应数据                                                                                      |
| ∟ id       | string   | 任务ID                                                                                      |
| ∟ status   | string   | 任务状态<br/>`running`：运行中 <br/>`failed`：失败<br/>`success`：成功                                  |
| ∟ view_url | string   | 总结查看地址                                                                                  |
| more       | object   | 更多的错误信息                                                                                    |

## 响应体示例

<!-- 为了保持文档格式统一，请使用 2 个空格作为 json 的缩进符 -->

```json
{
    "code": 0,
    "data":
    {
        "id": "123",
        "status": "finish",
        "view_url": "https://xxx.cn/xx"
    },
    "msg": "string"
}
```
