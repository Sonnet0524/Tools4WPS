# 在搬家任务中添加成员



**标签**：`邮箱搬家` 


## 请求说明
| 字段          | 值                  |
|--------------|---------------------|
| **<div style="white-space: nowrap;">请求地址</div>**  | https://openapi.wps.cn/v7/mail_migrations/{migration_id}/tasks          |
| **<div style="white-space: nowrap;">HTTP 方法</div>** | `POST`    |
| **<div style="white-space: nowrap;">接口描述</div>**  | 在搬家任务中添加成员。支持ex_user_id和user_id，需要在请求头中标明X-Kso-Id-Type使用 internal或external，默认是internal。注意：用户邮箱将被归入failed_list的情况及响应字段说明：(1)用户邮箱不属于搬家域名 - 响应仅包含user_id；(2)用户邮箱已在搬家任务中 - 响应包含user_id和email_address。 |
| **<div style="white-space: nowrap;">签名方式</div>**  | [KSO-1](https://open.wps.cn/documents/app-integration-dev/wps365/server/api-description/signature-description) |
| **<div style="white-space: nowrap;">限频策略</div>** | 无 |
| **<div style="white-space: nowrap;">权限要求</div>** | <div><div>查询和管理邮箱搬家(应用授权) `kso.mail_migration.readwrite`</div></div> |


## 请求头 (Headers)
| 属性名 | 类型 | 是否必填 | 描述 | 可选值 |
|-------|------|------|-------|-------|
| <div style="white-space: nowrap;">X-Kso-Id-Type</div> | `string` | 否 | 类型<br>  * **internal** - 内部<br>  * **external** - 外部<br> | `internal`, `external` 



## 路径参数 (Path)
| 属性名 | 类型 | 是否必填 | 描述 | 可选值 |
|-------|------|------|-------|-------|
| <div style="white-space: nowrap;">migration_id</div> | `string` | 是 | 搬家作业ID，填入current获取当前企业正在进行作业的搬家任务 | -
