# 会议室层级变更

## 事件描述

会议室层级资源变化时，包括 创建、修改、删除时触发此消息。

## 订阅说明

| **事件主体** | kso.meeting_room.level                                                                                       |
| :----------- | :--------------------------------------------------------------------------------------------- |
| **权限要求** | 查询会议室信息 `kso.meeting_rooms.read`                                                              |
| **解密方式** | 详见 [解密算法](/app-integration-dev/wps365/server/event-subscription/security-verification) |

### 事件体（解密前）

<!-- 为了保持文档格式统一：（1）参数说明中请使用中文全角标点符号；（2）中英文间请添加空格 -->

| **名称**       | **参数类型** | **说明**                                                       |
| :------------- | :----------- | :------------------------------------------------------------- |
| topic          | string       | 消息主题 `kso.meeting_room.level`                                            |
| operation      | string       | 消息变更动作<br/>`create`：创建；`update`：更新；`delete`：删除 |
| time           | integer      | 时间（秒为单位的时间戳）                                       |
| nonce          | string       | iv 向量（解密时使用）                                          |
| signature      | string       | 消息签名                                                       |
| encrypted_data | string       | 消息变更的加密字段                                             |

### 事件体示例（解密前）

<!-- 为了保持文档格式统一，请使用 2 个空格作为 json 的缩进符 -->

```json
{
  "topic": "kso.meeting_room.level",
  "operation": "create|update|delete",
  "time": 1704074400,
  "nonce": "71***********7",
  "signature": "w6**********6Q",
  "encrypted_data": "B7**********iA=="
}
```

### 事件体数据（解密后）

<!-- 为了保持文档格式统一：（1）参数说明中请使用中文全角标点符号；（2）中英文间请添加空格 -->

| **名称**      | **参数类型**      | **是否必带** | **说明**                        |
|:------------|:--------------|:---------|:------------------------------|
| company_id  | string        | 是        | 企业 id                         |
| room_levels | array[object] | 是        | 会议室层级变更信息数组                   |
| ∟ id        | string        | 是        | 会议室层级 id                      |
| ∟ name      | string        | 是        | 层级名称                          |
| ∟ parent_id | string        | 是        | 父亲层级id,当层级为企业根目录时，parent_id为0 |
| ∟ path      | array[string] | 是        | 会议室层级路径                       |

### 事件体数据示例（解密后）

<!-- 为了保持文档格式统一，请使用 2 个空格作为 json 的缩进符 -->

```json
{
    "company_id": "A*****6",
    "room_levels": [
        {
            "id": "2",          
            "name": "第二层级",  
            "parent_id": "1",     
            "path": [                 
                "1",
                "2"
            ]
        }
    ]
}
```
