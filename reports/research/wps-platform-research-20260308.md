## 五、权限体系说明

### 5.1 权限体系四维度

WPS开放平台采用四维度权限体系：

```
权限体系
├── 访问凭证（access_token）
│   ├── 应用身份验证
│   ├── 平台授权
│   └── Token有效期管理
├── API权限（scope）
│   ├── 接口调用权限
│   ├── 以应用为维度授予
│   └── 需要申请审批
├── 可用范围
│   ├── 定义可见用户
│   └── 控制应用使用范围
└── 数据权限
    ├── 数据访问范围
    ├── 通讯录数据权限
    └── 业务数据权限
```

### 5.2 权限申请流程

```
创建应用 → 申请API权限 → 配置数据权限 → 提交审核 → 审核通过 → 权限生效
```

**注意事项**：
- API权限申请后需要审核
- 数据权限配置后也需要审核
- 第三方企业应用由租户企业在安装时设置

### 5.3 数据权限规则

**通讯录数据权限规则**：
1. 部门权限包含子部门及下属员工
2. 修改用户部门需要同时拥有用户和目标部门权限
3. 修改部门父级需要同时拥有上级部门权限

## 六、认证授权流程

### 6.1 API调用流程（7步）

```
1. 创建应用
   └─ 获取APPID和APPKEY

2. 申请API权限
   └─ 在开发者后台申请所需接口权限

3. 获取访问凭证
   └─ 根据不同场景获取不同的access_token

4. 配置应用数据权限
   └─ 配置并提交审核

5. 设置IP白名单（可选）
   └─ 提升安全性

6. 了解通用参数
   └─ 掌握术语和概念

7. 了解通用错误码
   └─ 便于问题定位
```

### 6.2 访问凭证类型

不同的访问凭证代表不同的资源访问权限：
- **应用凭证**：代表应用身份
- **用户凭证**：代表用户授权
- **企业凭证**：代表企业资源访问

### 6.3 安全机制

- **签名验证**：API调用需要签名
- **IP白名单**：可设置IP访问限制
- **Token有效期**：访问凭证有过期时间
- **权限审批**：重要权限需要审核

## 七、关键技术要点

### 7.1 应用开发要点

**应用类型选择**：
- 企业内部使用 → 企业自建应用
- 对外提供服务 → 第三方企业应用
- 个人用户服务 → 第三方个人应用

**应用形态选择**：
- 后台服务 → 仅申请API权限
- 前端交互 → 配置相应能力（机器人、小组件等）

### 7.2 API调用要点

**调用前准备**：
1. 确认已申请对应API权限
2. 配置好数据权限
3. 获取正确的访问凭证
4. 了解接口参数要求

**调用注意事项**：
- 注意访问凭证有效期，及时刷新
- 遵守接口调用频率限制
- 正确处理错误码
- 做好异常处理

### 7.3 开发建议

**架构设计**：
- 建议使用配置文件管理APPID、APPKEY
- 封装API调用层，统一处理认证和错误
- 实现Token自动刷新机制
- 做好日志记录

**安全建议**：
- 不要在前端暴露APPKEY
- 使用签名机制
- 设置IP白名单
- 定期更换密钥
